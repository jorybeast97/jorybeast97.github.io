序言

近期冠状病毒肆虐，被困在家中，学校也没有给出一些网课的学习地址，所以自己就打算利用这一段时间来准备暑期实习的内容复习，由于自己是非科班学生，对于数据结构和算法的理解很浅，所以打算从《剑指offer》这本书开始复习，花了三天多的时间把里面的65道题看了一遍，有些重点的题自己也写了写，这里总结一下我觉得比较重要的一些题目和出现频率比较高的题目。

这次总结主要针对时间复杂度和空间复杂度两个方面考虑，由于自己能力有限，有些题目写的比较复杂琐碎，如果有更好的解决办法，请留言提醒。

数组中的重复数字

题目要求 : 在一个长度为 n 的数组里的所有数字都在 0 到 n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字是重复的，也不知道每个数字重复几次。请找出数组中任意一个重复的数字。

Input: {2, 3, 1, 0, 2, 5}

Output: 2

如果可以，请使用时间复杂度O(n)，空间复杂度为O(1)的解决方法。

由于数组中数据是在0 ~ n-1范围之内，所以最简单的方式就是使用排序然后依次比较即可。

但是由于题目要求，排序方法会超过时间限制，所以需要新的方法。

对于这种数组元素在 [0, n-1] 范围内的问题，可以将值为 i 的元素调整到第 i 个位置上进行求解。本题要求找出重复的数字，因此在调整过程中，如果第 i 位置上已经有一个值为 i 的元素，就可以知道 i 值重复。

从尾到头打印链表

题目要求 : 从尾部到头部逆序打印单向链表

非常经典的面试题目，能够解决的方法也比较多，我比较喜欢使用递归的方法和栈，都是解决这个问题比较好的途径。

重建二叉树

题目要求 : 输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

题解来自牛客网 : 皮卡丘的博客

根据中序遍历和前序遍历可以确定二叉树，具体过程为：

  1. 根据前序序列第一个结点确定根结点
  2. 根据根结点在中序序列中的位置分割出左右两个子序列
  3. 对左子树和右子树分别递归使用同样的方法继续分解

例如：
前序序列{1,2,4,7,3,5,6,8} = pre
中序序列{4,7,2,1,5,3,8,6} = in

  1. 根据当前前序序列的第一个结点确定根结点，为 1
  2. 找到 1 在中序遍历序列中的位置，为 in[3]
  3. 切割左右子树，则 in[3] 前面的为左子树， in[3] 后面的为右子树
  4. 则切割后的左子树前序序列为：{2,4,7}，切割后的左子树中序序列为：{4,7,2}；切割后的右子树前序序列为：{3,5,6,8}，切割后的右子树中序序列为：{5,3,8,6}
  5. 对子树分别使用同样的方法分解

用两个栈实现队列

题目要求 : 用两个栈来实现一个队列，完成队列的 Push 和 Pop 操作。

实现思路很简单，将数据压入一个栈后，然后弹出再压入另一个栈，此时从第二个栈中取出的，顺序就是队列顺序。

斐波那契数列以及经典跳台阶问题

题目要求 : 求斐波那契数列的第 n 项，n <= 39。

递归解法

虽然递归解法是最简洁最容易处理的，但是在递归的过程中，会重复计算许多相同的数值，这种情况会浪费非常多的资源，动态规划可以将递归思想反过来递推，自底向上的思想即可。

只需要将我们每次计算的结果都储存起来，每次用的时候读取缓存即可。

青蛙跳台阶同理，每次青蛙能够跳一个或者两个台阶，由此可以推导出

旋转数组

题目要求 : 给定一个数组，将数组中的元素向右移动 k 个位置，其中 k 是非负数。

输入: [1,2,3,4,5,6,7] 和 k = 3
输出: [5,6,7,1,2,3,4]
解释:
向右旋转 1 步: [7,1,2,3,4,5,6]
向右旋转 2 步: [6,7,1,2,3,4,5]
向右旋转 3 步: [5,6,7,1,2,3,4]

使用额外空间

最简单的方法就是使用额外空间来完成。

但是这道题在大部分情况下，都是要求不能使用额外空间的。

空间复杂度O(n)的解法

给出数组 [1, 2, 3, 4, 5, 6, 7, 8, 9 ] ，假如k = 3

第一次交换 [9，8，7，6，5，4，3，2，1]

第二次交换 [7，8，9，6，5，4，3，2，1]

第三次交换 [7，8，9，1，2，3，4，5，6]

那么就需要一个函数能够对数组进行翻转的操作。

旋转数组的最小值

题目要求 : 把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。输入一个非递减排序的数组的一个旋转，输出旋转数组的最小元素。

我们可以举个例子 [7,8,9,1,2,3,4,5,6],这是一个旋转数组，在这个数组中，能够发现，当前数的下一个数应该比自己大，如果小于自己，该数就是旋转数组中最小的数。

剪绳子

剪绳子问题是非常经典的动态规划问题，由此可以引申出很多关于DP的题目思路。

题目要求 : 把一根绳子剪成多段，并且使得每段的长度乘积最大。

暴力法

暴力法应该是能想到最简单的解法，我们需要记住每一种结果然后进行比较，取最大的值即可。为了实现这种方法我们需要推导一个公式。但是这种方法会在Leetcode中导致OJ，尽量不要使用。

动态规划

虽然暴力法能够计算出最后结果，但是和以往递归操作一样，会计算很多很多的重复数据。所以需要将之前计算的结果保存在数组中。
