Mybatis Lite配合Mybatis Plus快速开发
发表于 2020-03-09| 更新于 2020-03-09|工具

网课开展，发现并没有想象中的轻松，相反很多在课堂上能快速完成的事情通过网络需要浪费好多时间，所以最近也就没怎么给自己充电，但是总结了一些自己用着比较顺手的IDEA快速开发的插件。

Mybatis Lite + Mybatis Plus

大部分Java工程师选择ORM框架的时候都是使用Mybatis，我想原因有二

  • Mybatis成熟度高,业内用户量大,效率也不错
  • Mybatis能够手写SQL语句,对于SQL优化以及多表操作有着非常便捷的优势

但是由于Mybatis的XML Mapper配置十分繁琐，所以很多时候在开发效率上要和其他的ORM框架有所差距，印象最深的一次是有一个四十多个字段的表，当时为了写Model和Mapper.xml，几乎用了快半天的时间。很多大佬也意识到了这个情况，所以出现了例如tk.mybats和Mybatis Plus这种增强框架，能够通过实现接口的方式完成对单表的简单操作。

但是我自己在学习的过程中，还是发现了一个问题，当我们需要学习一个依赖数据库和DAO层的项目时，即便是使用Mybatis Plus，效率依旧有些慢，因为还是要自己写Model，自己去写Mapper，虽然这个Mapper很简单。程序员永远是最懒惰的人，所以我发现可以使用Mybats Lite Plugin和Mybatis Plus配合使用。由于Mybatis Plus是一款Mybatis的增强型框架，这就意味着能够无侵入的增强Mybatis，使用起来会非常顺滑。

而Mybatis Lite是一款Mybatis Generator插件，能够为我们的数据表生成对应的Model、Mapper、Service和Controller等服务层，这两者搭配使用，可以完成对于单表除建表以外的零操作，而对于多表我们只需要在相应的Mapper中写Sql语句即可，使用起来非常方便。

Mybatis Lite下载 > File > Settings > Plugins > 搜索Mybatis Lite > Install and Restart

安装完成后，在Tools中能够找到Mybatis Lite，其中有Generator和Temple

不需要再向以往的工程一样，手写大量重复的工作。运行一下

自动生成后的MangerUserPo类以及相关的套件已经完成，当然，这之中有很多不太完美的地方，比如数据库中的tinyInt我们更加期望对应boolean，这些细节问题我们可以在使用到的时候更改一下即可，这样我们就能够简单完成单表的各种操作，同时如果需要使用复杂的语句，可以在Mapper中编写即可，试一下查询权限的多表联查。

尝试一下能够完成查询。使用起来还是非常方便的。
