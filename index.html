Select NIO概述

首先我们来看下在Linux文档中，该文档中给出了很多关于Linux下Select中的各种信息，将这些信息理解后，基本上对于Linux下的Select NIO模型就有了一个很完善的掌握。

select(2)——Linux manual page

开头给出了几个重点的方法，每个方法在文档中都有很详细的解释，首先关注一个重点的方法select。

在文档中，对于select方法描述如下。

select() allows a program to monitor multiple file descriptors,
waiting until one or more of the file descriptors become “ready” for
some class of I/O operation (e.g., input possible). A file
descriptor is considered ready if it is possible to perform a
corresponding I/O operation (e.g., read(2), or a sufficiently small
write(2)) without blocking.

select() can monitor only file descriptors numbers that are less than
FD_SETSIZE; poll(2) and epoll(7) do not have this limitation. See
BUGS.

执行select()方法后，该进程将监控多个文件描述符(fd)，将这些文件描述符存入一个整形的数组中，并将该数组拷贝进入内核空间，然后该线程将阻塞。

当IO时间就绪,发生或者超时的时候，内核空间会轮询这个fd_set，找到就绪的事件，将这些事件再从内核空间拷贝到用户空间，此时select()不再阻塞，执行之后的代码。

Bloom Filter与Redis缓存穿透

发表于 2020-06-17 分类于 技术思考

什么是布隆过滤器

布隆过滤器（英语：Bloom Filter）是1970年由布隆提出的。它实际上是一个很长的二进制向量和一系列随机映射函数。布隆过滤器可以用于检索一个元素是否在一个集合中。它的优点是空间效率和查询时间都远远超过一般的算法，缺点是有一定的误识别率和删除困难。

布隆过滤器的使用场景比较广泛，其代表性场景有IP地址过滤，缓存击穿避免。而这两个场景都指向了一个问题。

🏳‍🌈 如何判断一个元素是否在集合当中 ?

实际上这个问题有太多的结局方案，只需要一个容器和一种查询算法，无论使用数组+二分查找，或者使用二叉搜索树，红黑树，或者使用跳表，同样能够使用HashSet来处理。但是随着数据量的增大，这个集合将变得无比巨大，而这种内存消耗增长所带来的收益却并不是线性增长。

由于我们只关心某个数据是否在该集合中，并不需要数据的具体信息，所以我们可以将某个关注的字段进行储存，其他的字段不做处理即可。这种思维就是Bloom Filter的思维，而布隆过滤器做的更加彻底。

布隆过滤器的原理是，当一个元素被加入集合时，通过K个Hash函数将这个元素映射成一个位数组(bit)中的K个点，把它们置为1。检索时，我们只要看看这些点是不是都是1就（大约）知道集合中有没有它了：如果这些点有任何一个0，则被检元素一定不在；如果都是1，则被检元素很可能在。

TCP SYN队列与SYN泛洪攻击

发表于 2020-06-15 分类于 网络

TCP半连接队列和全连接队列

TCP通讯是通过三次握手来保证通讯的可靠性，三次握手相关的知识网上的文章很多，这里就不花费篇幅来总结。

由于三次握手都是通过网络来进行的，所以三次握手之间都有一个时间间隔，那么在未完成三次握手之前，就需要维护两个队列来存放这些不同状态的连接。

  • 半连接队列，也称 SYN 队列
  • 全连接队列，也称 accepet 队列

当客户端发送第一次发送SYN后，服务器接收到该TCP的SYN请求时，会将该TCP连接存放进入SYN队列，当第三次握手完成时，服务端会将保存在SYN队列中的TCP连接移入道accept队列中。

可想而知，不论是SYN队列还是Accept队列都会有自己的上限，无论设置多大，当超过限制后，内核可以直接丢弃或者返回RST包。

synchronized等待池与锁池

发表于 2020-06-14 分类于 Java基础技术

偶然中看到了synchronized的等待池和锁池的概念，在之前学习synchronized的时候并没有学习到这个部分，因此抽一点时间总结一下。

实际上理解锁池和等待池可以从一个问题出发。

✔ 为什么wait()，notify(),notifyAll()方法在Object类中 ?

在Java中所有的类都是Object类的子类，也就是说，所有的对象，归根结底就是Object对象。而Java中的锁可以分为对象锁和类锁，每一个对象都有成为一把锁的潜质。

在对象的Markword中会记录对象锁相关的信息，例如锁等级,当前锁持有者等信息。我们来看下Doc中的wait方法的介绍。

Causes the current thread to wait until either another thread invokes the notify() method or the notifyAll() method for this object, or a specified amount of time has elapsed.

将当前线程所持有的锁放弃，当其他持有该锁的线程调用该锁的notify(),notifyAll()方法时会被唤醒。这就是为什么一定要将这些方法设计在Object类中，因为线程的状态是需要通过锁来操作并存放在锁中。
