在输入一个URL后浏览器发生了什么?
发表于 2019-12-21| 更新于 2019-12-21|网络
阅读量:

序言

近来一只在考试，Github也没什么产出，好几天每碰过代码了，总算是到了学期尾声，剩余时间能安安心心做做自己的事情了。

在输入一个URL后浏览器发生了什么?

这个问题一直是个热门的面试题，究其根本就是能从整体上了解一个学生的知识体系，很多时候自己在开发项目的过程中并没有仔细想过，只是模糊的直到输入一个URL，发送一个HTTP数据报，然后请求到达服务器，接受请求处理完成后响应给服务器，然后前端页面完成渲染。可是如果根据细节来说，就手足无措了，所以打算通过这道面试题，好好的把这些只是梳理一下。

大致流程

本文的步骤是建立在，请求的是一个简单的 HTTP 请求，没有 HTTPS、HTTP2、最简单的 DNS、没有代理、并且服务器没有任何问题的基础上，尽管这是不切实际的。

按照自己的思维，我想可以将其分为以下几个流程

  1. URL查询
  2. DNS解析
  3. TCP请求
  4. 发送请求
  5. 处理请求
  6. 接收响应
  7. 渲染页面
