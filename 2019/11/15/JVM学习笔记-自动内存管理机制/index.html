JVM学习笔记:自动内存管理机制
发表于 2019-11-15| 更新于 2019-11-16|JVM学习笔记
阅读量:

JVM内存区域划分图

Java虚拟机在执行Java程序中会把所管理的内存分给为不同个数据区域，这些区域有着各自的用途以及创建和销毁的时间。

程序计数器

程序计数器是内存中的一小块内容，可以看作当前程序运行的行数指示器

在Java中，多线称的实现是通过线程的轮流切换并分配CPU处理器的时间片，在任何一个时刻，只有一个程序能够被CPU(一个核心)所执行，为了保证多个程序在切换后能让CPU正确按照顺序执行之后的操作，所以每一个线程都需要一个程序计数器来记录CPU当前所执行到的行数。各条线程之间互不影响，为”线程私有”。

Java虚拟机栈

Java虚拟机栈是存放Java方法的内存模型，同样是线程私有，其生命周期与线程相同。

每当方法执行的时候,都会创建一个栈帧(Stack Frame)，用于存储局部变量，操作数栈，动态链接，方法出口等模型。每一个方法从调用到完成都是一个栈帧的入栈与出栈过程。

在常见的”栈内存”称呼中，一般来说就是指虚拟机栈中的局部变量表。因为该表中保存着一个方法的各种变量信息。局部变量表存放了编译器可知的各种基本数据类型，对象引用和返回值地址类型(指向一条字节码指令的地址)。

Java虚拟机栈的异常

虚拟机规范中，这个区域定义了两个异常情况：

  • StackOverflowError : 当线程请求的栈深度大于虚拟机所允许的深度
  • OutOfMemoryError : 当拓展时无法申请到足够的内存

本地方法栈

本地方法栈几乎与虚拟机栈相同，只不过虚拟机栈为JVM执行Java方法，而本地方法栈则为虚拟机执行的时native方法。由于JVM规范中对本地方法栈的要求十分宽松，HotSpots甚至将虚拟机栈和本地方法栈合二为一。

与虚拟机栈相同，本地方法栈也会抛出StackOverFlowError，OutOfMemoryError的异常

堆

Java堆是用来存放内存实例的区域，该区域被所有线程所共享，在虚拟机启动时创建。

方法区

方法区时存储虚拟机加载的类信息，常量，静态变量编译后的代码等数据的线程共享的区域。

在早期HotSpots中将其成为永久代，意思是该区的对象回收较少，很多将一直存活，后期被元空间所替代。
