社交媒体 & 云服务器 & Github

自2018年初搭建了第一个个人博客后,就一直在学习并且不断折腾自己的博客内容。从CSDN，掘金等社交媒体，到自己购买云服务器与域名，再到现在的Github page，一直也在摸索学习，虽然不确定以后还会不会更换(大概率不会了，换来换去太累了)，记录下各个平台的一些优缺点以供大家所选择吧。

主流社交媒体

在国内的话像博客园,CSDN这类技术社区实际上还是比较火热的，虽然很多时候上面的内容都是来回抄袭，有时候也是看的很无奈，但不得不承认，在这个大环境下，这两大社区依旧是比较方便的，尤其对于刚刚学习编程并且想要记录自己的一些学习历程的同学来说，上手难度极低，曝光度也不错，是个好一点的选择。

二者中我更加推荐博客园一点，自己可以定制页面，同时界面清爽一点，看上去没那么CSDN那么凌乱。

云服务器部署自己的站点

先挂上自己目前还没到期的博客地址吧:

一只草莓的旅行

老实讲我真的很喜欢这个站点的风格，而使用WP部署也十分便捷，同时对于学习一段时间的同学来说，使用Linux + SSH的方式也能巩固一定的基础，但是对我来说最大的问题在于。我对于托管在云上的东西安全感都不太好，我更希望文章在我本机内都有一个备份，这样哪怕是哪天云服务到期或者重装后，我还能快速部署。

Github Pages

这个博客使用的Github page + Hexo的方式搭建的，总体下来也就不到两天的时间，部署非常便捷，好评。

原理这里不多说，相信阅读这篇文章的同学都理解了Github page的部署原理.

我觉得Github page最好的一点在于，它的提交内容能够和github个人账户的commit挂钩，之前我的博客产出不少，但是在Github上却是一片空白，看着别的大佬绿绿的一片，总有些羡慕，所以这次使用了这个。优点很多，我总结下吸引我的。

  • 主题多，无缝切换
  • 本地编译上传，数据安全
  • 能够提升Github的commit

可能不是最好的，但却是最适合我的！
